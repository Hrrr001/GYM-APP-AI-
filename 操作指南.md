# 个性化健身训练计划应用操作指南

## 项目简介

这是一个基于Flutter和Spring Boot开发的个性化健身训练计划应用，旨在为用户提供定制化的健身指导和管理服务。应用集成了AI技术，能够根据用户的个人情况生成专属的训练计划和饮食建议，帮助用户更科学、有效地进行健身训练。

## 功能特性

### 1. 用户管理
- 用户注册与登录
- 个人档案管理（基本信息、健身目标、身体状况等）
- 身体测量记录（身高、体重、体脂率等）

### 2. 训练管理
- 训练计划生成与管理
- 训练记录追踪
- 运动详情查看

### 3. 饮食管理
- 饮食记录
- AI饮食建议

### 4. AI健身助手
- 回答健身相关问题
- 生成动作详情
- 推荐适合的运动
- 分析训练数据
- 生成个性化训练计划
- 提供饮食建议

## 技术栈

### 前端
- **框架**：Flutter
- **语言**：Dart
- **状态管理**：Flutter内置状态管理
- **网络请求**：http package

### 后端
- **框架**：Spring Boot 2.7.18
- **语言**：Java
- **数据库**：MySQL
- **ORM**：JPA
- **安全**：Spring Security

## 安装与配置指南

### 系统要求

#### 后端
- Java 8 或更高版本
- Maven 3.6 或更高版本
- MySQL 5.7 或更高版本

#### 前端
- Flutter SDK 3.0 或更高版本
- Dart SDK 2.17 或更高版本
- 支持的平台：Android、iOS、Windows、macOS、Linux

### 后端配置与启动

1. **数据库配置**
   - 创建MySQL数据库，名称建议为`gym_app`
   - 修改`GymApp/backend/src/main/resources/application.properties`文件，配置数据库连接信息：
   
   ```properties
   spring.datasource.url=jdbc:mysql://localhost:3306/gym_app?useSSL=false&serverTimezone=UTC
   spring.datasource.username=root
   spring.datasource.password=your_password
   spring.jpa.hibernate.ddl-auto=update
   ```

2. **构建与启动**
   - 进入`GymApp/backend`目录
   - 执行以下命令构建项目：
   
   ```bash
   mvn clean package
   ```
   
   - 执行以下命令启动后端服务：
   
   ```bash
   java -jar target/backend-0.0.1-SNAPSHOT.jar
   ```
   
   - 后端服务将在 http://localhost:8080 运行

### 前端配置与启动

1. **依赖安装**
   - 进入`GymApp/frontend`目录
   - 执行以下命令安装依赖：
   
   ```bash
   flutter pub get
   ```

2. **启动应用**
   - 执行以下命令启动前端应用：
   
   ```bash
   flutter run
   ```
   
   - 或者在IDE中运行（如Android Studio、VS Code）

3. **构建发布版本**
   - Android：
   
   ```bash
   flutter build apk
   ```
   
   - iOS：
   
   ```bash
   flutter build ios
   ```
   
   - Windows：
   
   ```bash
   flutter build windows
   ```

## 详细使用指南

### 1. 用户注册与登录

#### 注册流程
1. 打开应用，点击「注册」按钮
2. 输入用户名、密码、邮箱
3. 点击「注册」按钮完成注册

#### 登录流程
1. 打开应用，在登录页面输入用户名和密码
2. 点击「登录」按钮进入主界面

### 2. 个人档案管理

#### 创建个人档案
1. 登录后，系统会提示完善个人档案
2. 填写姓名、性别、年龄、身高、体重等基本信息
3. 选择健身水平、健身目标
4. 填写伤病情况和设备条件
5. 点击「保存」按钮完成档案创建

#### 修改个人档案
1. 在主界面点击「个人档案」选项
2. 修改相应信息
3. 点击「保存」按钮完成更新

### 3. 身体测量记录

#### 添加身体测量数据
1. 在主界面点击「身体测量」选项
2. 点击「添加测量」按钮
3. 填写测量日期、身高、体重、体脂率、肌肉量等数据
4. 点击「保存」按钮完成记录

#### 查看身体测量历史
1. 在主界面点击「身体测量」选项
2. 查看历史测量记录
3. 可以通过图表查看身体数据变化趋势

### 4. 训练计划管理

#### 查看训练计划
1. 在主界面点击「训练计划」选项
2. 查看系统生成的训练计划列表
3. 点击计划名称查看详细内容

#### 生成个性化训练计划
1. 在主界面点击「AI助手」选项
2. 选择「生成训练计划」功能
3. 等待AI生成计划
4. 查看并保存生成的计划

#### 执行训练计划
1. 选择一个训练计划
2. 按照计划中的训练内容进行锻炼
3. 完成训练后，点击「打卡」按钮记录训练情况

### 5. 训练记录管理

#### 添加训练记录
1. 在主界面点击「训练记录」选项
2. 点击「添加记录」按钮
3. 填写训练日期、时长、消耗卡路里
4. 添加训练详情（运动名称、组数、次数、重量）
5. 点击「保存」按钮完成记录

#### 查看训练记录
1. 在主界面点击「训练记录」选项
2. 查看历史训练记录列表
3. 点击记录查看详细内容

### 6. 饮食管理

#### 添加饮食记录
1. 在主界面点击「饮食记录」选项
2. 点击「添加记录」按钮
3. 填写日期、食物名称、卡路里、蛋白质、碳水化合物、脂肪等信息
4. 点击「保存」按钮完成记录

#### 获取AI饮食建议
1. 在主界面点击「AI助手」选项
2. 选择「饮食建议」功能
3. 等待AI生成饮食建议
4. 查看并应用建议

### 7. AI健身助手使用

#### 回答健身问题
1. 在主界面点击「AI助手」选项
2. 在聊天框中输入健身相关问题
3. 点击「发送」按钮
4. 查看AI的回答

#### 推荐适合的运动
1. 在主界面点击「AI助手」选项
2. 选择「推荐运动」功能
3. 等待AI生成推荐
4. 查看推荐的运动列表

#### 分析训练数据
1. 在主界面点击「AI助手」选项
2. 选择「分析训练数据」功能
3. 等待AI分析数据
4. 查看分析结果和建议

## 常见问题与解决方案

### 1. 无法登录
- **问题**：输入正确的用户名和密码后无法登录
- **解决方案**：
  - 检查用户名和密码是否正确
  - 检查后端服务是否运行
  - 检查网络连接是否正常

### 2. 无法添加数据
- **问题**：无法添加训练记录、饮食记录或身体测量数据
- **解决方案**：
  - 检查所有必填字段是否填写
  - 检查后端服务是否运行
  - 检查网络连接是否正常

### 3. AI功能无响应
- **问题**：使用AI功能时没有响应
- **解决方案**：
  - 检查网络连接是否正常
  - 检查后端服务是否运行
  - 尝试重新发送请求

### 4. 应用崩溃
- **问题**：应用突然崩溃
- **解决方案**：
  - 重启应用
  - 检查是否为最新版本
  - 清除应用缓存

### 5. 数据不显示
- **问题**：数据记录后不显示
- **解决方案**：
  - 刷新页面
  - 检查网络连接
  - 检查后端服务是否运行

## 系统架构与技术说明

### 1. 系统架构

- **前端**：Flutter应用，负责用户界面和交互
- **后端**：Spring Boot应用，负责业务逻辑和数据处理
- **数据库**：MySQL，存储用户数据和业务数据
- **AI服务**：集成AI能力，提供智能建议和分析

### 2. 核心API接口

#### 用户相关
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `POST /api/users/profile` - 创建用户档案
- `PUT /api/users/profile` - 更新用户档案
- `GET /api/users/profile/{userId}` - 获取用户档案

#### AI相关
- `POST /api/ai/question` - 回答健身相关问题
- `POST /api/ai/exercise/details` - 生成动作详情
- `POST /api/ai/exercise/recommend` - 推荐适合的运动
- `POST /api/ai/plan/generate` - 生成训练计划
- `POST /api/ai/workout/analyze` - 分析训练数据
- `POST /api/ai/nutrition/suggestion` - 生成饮食建议

#### 训练相关
- `GET /api/exercises` - 获取运动列表
- `GET /api/exercises/{id}` - 获取运动详情
- `GET /api/plans` - 获取训练计划列表
- `GET /api/plans/{id}` - 获取训练计划详情
- `POST /api/plans` - 创建训练计划
- `PUT /api/plans/{id}` - 更新训练计划
- `DELETE /api/plans/{id}` - 删除训练计划

#### 饮食相关
- `GET /api/nutrition/records` - 获取饮食记录
- `POST /api/nutrition/records` - 添加饮食记录
- `PUT /api/nutrition/records/{id}` - 更新饮食记录
- `DELETE /api/nutrition/records/{id}` - 删除饮食记录

#### 训练记录相关
- `GET /api/workouts` - 获取训练记录列表
- `GET /api/workouts/{id}` - 获取训练记录详情
- `POST /api/workouts` - 添加训练记录
- `PUT /api/workouts/{id}` - 更新训练记录
- `DELETE /api/workouts/{id}` - 删除训练记录

### 3. 数据库设计

#### 主要表结构

1. **users** - 用户表
   - id: 主键
   - username: 用户名
   - password: 密码（加密存储）
   - email: 邮箱
   - created_at: 创建时间

2. **user_profiles** - 用户档案表
   - id: 主键
   - user_id: 外键，关联users表
   - name: 姓名
   - gender: 性别
   - age: 年龄
   - height: 身高
   - weight: 体重
   - fitness_level: 健身水平
   - fitness_goal: 健身目标
   - injuries: 伤病情况
   - equipment_condition: 设备条件

3. **exercises** - 运动表
   - id: 主键
   - name: 运动名称
   - description: 运动描述
   - muscle_group: 目标肌群
   - difficulty: 难度等级

4. **training_plans** - 训练计划表
   - id: 主键
   - user_id: 外键，关联users表
   - name: 计划名称
   - goal: 训练目标
   - duration: 计划时长（周）
   - created_at: 创建时间

5. **plan_details** - 训练计划详情表
   - id: 主键
   - plan_id: 外键，关联training_plans表
   - day_of_week: 周几
   - exercises: 当天运动（JSON格式）

6. **workout_records** - 训练记录表
   - id: 主键
   - user_id: 外键，关联users表
   - date: 训练日期
   - duration: 训练时长（分钟）
   - calories: 消耗卡路里

7. **workout_details** - 训练记录详情表
   - id: 主键
   - record_id: 外键，关联workout_records表
   - exercise_name: 运动名称
   - sets: 组数
   - reps: 次数
   - weight: 重量

8. **food_records** - 饮食记录表
   - id: 主键
   - user_id: 外键，关联users表
   - date: 记录日期
   - food_name: 食物名称
   - calories: 卡路里
   - protein: 蛋白质
   - carbs: 碳水化合物
   - fat: 脂肪

9. **body_measurements** - 身体测量表
   - id: 主键
   - user_id: 外键，关联users表
   - date: 测量日期
   - height: 身高
   - weight: 体重
   - body_fat: 体脂率
   - muscle_mass: 肌肉量

## 性能优化建议

1. **数据库优化**
   - 为常用查询字段添加索引
   - 合理设计表结构，避免冗余字段
   - 定期清理无用数据

2. **后端优化**
   - 使用缓存减少数据库查询
   - 优化API响应时间
   - 合理使用线程池处理并发请求

3. **前端优化**
   - 减少不必要的重绘和重排
   - 优化图片资源
   - 使用懒加载减少初始加载时间

## 安全注意事项

1. **数据安全**
   - 密码使用BCrypt加密存储
   - 敏感数据传输使用HTTPS
   - 定期备份数据库

2. **应用安全**
   - 防止SQL注入攻击
   - 防止XSS攻击
   - 防止CSRF攻击
   - 合理设置API访问权限
